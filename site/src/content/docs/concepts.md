## 仓库索引

`leo repo reindex` 扫描配置的根目录，识别 Git 仓库并将绝对路径和 Git 元数据写入本地 SQLite。选择器从索引读取结果，因此无需在每次打开时重新扫描全部目录。记录按绝对路径更新；当前不会自动删除已经不在磁盘上的旧记录。

## 日志安全边界

日志查看器只会从解析出的日志根目录建立目录，并在每次读取时重新验证允许范围。它跳过隐藏文件、符号链接、压缩文件、二进制内容和不支持的后缀。搜索有文件大小、结果数、并发和时间边界；跟随模式会识别截断、替换、删除和轮转。

HTTP 服务默认只绑定 loopback。浏览器先使用一次性 bootstrap token，再使用内存 session cookie；服务结束时状态消失。绑定非 loopback 地址会把临时工作台暴露给网络，只应在可信开发网络中使用。

## 本地存储与隐私

仓库元数据保存在本地 SQLite。日志内容仍以原文件为唯一数据源：查看器不复制日志、不建立持久全文索引，也不保存查询、token、session 或 UI 状态。`join` 只在用户确认后把结果写入系统剪贴板。
